Для получения артефактов (вместо класса можно использовать наше решение) можно использовать контроллеры, которые будут по конфигу, задаваемому из json выдавать результат
Для реализации такого подхода необходимо получать информацию из адаптеров на сервере. Для этого необходимо реализовать передачу правила информации с фронта на бэк
Это версия 2 диаграмм и схем. Для схемы рецепта надо добавить что-то аля id пользователя, для которого нужна инфа.

Набор функций-игредиентов включает в себя функции расчёта, функции агргации, функию возварщения ранее подсчитанных элементов.

Наверное, надо будет даль пользователю писать функции для получения определённых значений из схем, потому что запрогать это в общем виде будет сложно.
Это можно сделать через редактируемые блоки

Передавать источники на фронт можно как строки (потом через репозиторий коннектить их с адаптерами)
Передавать функции на фрокт можно как строки (потом через библиотеку функций коннектить их с функциями)
Рецепты формировать через встроенные в приложение функции, поля схем и кастомные json с указанием способа агрегации (редактируемый блок)
